## Overview

`WebsocketAPIClient` is a minimal, promise-driven wrapper around `WebsocketClient`. It lets you submit, cancel, and amend orders over WebSocket using async/await, without managing the underlying connection lifecycle yourself.

Each method sends a request over the appropriate private or business WebSocket connection and returns a promise that resolves when the matching response arrives from OKX.

<Note>
  `WebsocketAPIClient` uses `WebsocketClient` internally. You can access the underlying client at any time via `wsApiClient.getWSClient()` to subscribe to channels, listen to events, or call `sendWSAPIRequest()` directly.
</Note>

```typescript
import { WebsocketAPIClient } from 'okx-api';

const client = new WebsocketAPIClient({
  accounts: [
    {
      apiKey: 'YOUR_API_KEY',
      apiSecret: 'YOUR_API_SECRET',
      apiPass: 'YOUR_API_PASSPHRASE',
    },
  ],
});

// Place an order
const result = await client.submitNewOrder({
  instId: 'BTC-USDT',
  tdMode: 'cash',
  side: 'buy',
  ordType: 'market',
  sz: '0.01',
});
```

## Constructor

```typescript
new WebsocketAPIClient(
  options?: WSClientConfigurableOptions & Partial<WSAPIClientConfigurableOptions>,
  logger?: DefaultLogger
)
```

<ParamField body="options" type="object">
  Accepts all [`WSClientConfigurableOptions`](/api-reference/websocket-client#constructor-options-wsclientconfigurableoptions) fields (credentials, market, demo trading, etc.) plus the additional options below.
</ParamField>

<ParamField body="logger" type="DefaultLogger">
  Optional custom logger.
</ParamField>

### Additional options (`WSAPIClientConfigurableOptions`)

<ParamField body="attachEventListeners" type="boolean" default="true">
  When `true`, the client automatically attaches default event listeners that log connection lifecycle events (`open`, `reconnect`, `reconnected`, `authenticated`, `exception`) to the console.

  Set to `false` if you want to handle all events yourself via `getWSClient().on(...)`.
</ParamField>

## Utility methods

#### `getWSClient()`

```typescript
getWSClient(): WebsocketClient
```

Returns the underlying `WebsocketClient` instance. Use this to subscribe to channels, listen to raw events, or call `sendWSAPIRequest()` directly.

---

#### `connectWSAPI()`

```typescript
connectWSAPI(): Promise<unknown[]>
```

Pre-warms both the private and business WebSocket connections and authenticates them. Call this before your first order request to reduce initial latency.

---

#### `setTimeOffsetMs(offsetMs)`

```typescript
setTimeOffsetMs(offsetMs: number): void
```

Manually set the time offset (ms) between the local clock and OKX servers. Delegates to the underlying `WebsocketClient`.

## Order management methods

All order methods require authentication. Credentials must be provided in the constructor options.

---

### `submitNewOrder(params)`

```typescript
submitNewOrder(
  params: WSAPIPlaceOrderRequestV5
): Promise<WSAPIResponse<[OrderResult], 'order'>>
```

Place a single new order via the WebSocket API.

<ParamField body="params" type="WSAPIPlaceOrderRequestV5" required>
  Order parameters. Key fields include:

  - `instId` (string, required) ‚Äî Instrument ID, e.g. `'BTC-USDT'`.
  - `tdMode` (string, required) ‚Äî Trade mode: `'cash'`, `'cross'`, or `'isolated'`.
  - `side` (string, required) ‚Äî `'buy'` or `'sell'`.
  - `ordType` (string, required) ‚Äî Order type: `'market'`, `'limit'`, `'post_only'`, `'fok'`, `'ioc'`, etc.
  - `sz` (string, required) ‚Äî Order size.
  - `px` (string) ‚Äî Price. Required for limit orders.
  - `ccy` (string) ‚Äî Margin currency. Required for cross-margin on non-USDT pairs.
  - `clOrdId` (string) ‚Äî Client-assigned order ID.
  - `tag` (string) ‚Äî Order tag.
</ParamField>

**OKX docs:** [WS place order](https://www.okx.com/docs-v5/en/#order-book-trading-trade-ws-place-order)
